semantic - 의미론적인 태그

현시대 웹사이트들은 전부 semantic 태그를 사용한다.
semantic태그란 무엇인가를 설명하기에 앞서,
웹사이트를 만들때, 공간을 구분짓기위해 div 나 span 태그를 사용해 만들어도된다.
하지만, div 나 span 만을 사용한다면, 웹사이트가 완성되어갈쯔음에는
수많은 div와 span이 만들어지게 되므로 헷갈리고 내가 찾는 div가 어디인지 오래걸리게된다.
그리하여, 등장한것이 semantic 태그이다.


HTML5 의 semantic 구조 태그 호근형의 집중공부부분
태그 이름 설명
header 헤더를의미
nav 네비게이션을 의미
aside 사이드에 위치하는 공간을 의미
section 여러 중심내용을 감싸는 공간을 의미
article 글자가 많이 들어가는 부분을 의미
footer 푸터를 의미


간단한 웹사이트를 만들어볼텐데, div와 span만을 이용하여 만들어보는것과,
semantic 태그를 이용하여 만들어보는것의 차이점을 느껴보세요.
